Documentation Roadmap  

The Parasolid library is supplied with a comprehensive set of documentation, in both HTML format (for online browsing) and PDF format (so that you can print out any part of the documentation easily). The HTML version that you are currently looking at contains extensive links that make it easy to navigate to the relevant information, as well as a full text search feature.

The Parasolid documentation set consists of the following manuals:

What's New in Parasolid: For existing customers, a guide to the new features in the latest version of Parasolid. As well as describing functional enhancements to Parasolid, this guide contains a complete listing of all new and updated interfaces in the API.

Overview of Parasolid: Adetailed overview of the functionality available in Parasolid. If you are new to Parasolid and want to find out what it can do, then this manual will tell you. The Parasolid Overview does not describe any technical details about how you write Parasolid code.

Installation Notes: A guide to installing Parasolid on your system, together with build details such as compiler settings for each of the platforms it supports.

Functional Description: A comprehensive guide to the functionality provided by the Parasolid library. This manual should be your first stop when you want to learn about how to use any aspect of the functionality provided by Parasolid.

PK Reference: A complete reference to every function, structure, and token available in the Parasolid API. This manual is the ultimate Parasolid programmer's reference. (Only available in HTML)

Downward Interfaces: A guide to the Parasolid Downward Interfaces. These modules form part of your application and control all interaction between Parasolid and the operating system, handling tasks such as file and memory management.

Kernel Interface Driver: The Kernel Interface Driver (KID) is a platform-independent application available with Parasolid that provides a command line interface to the Parasolid API, allowing you to build, view, and manipulate Parasolid models on screen. User interaction with KID is via a LISP interpreter, allowing you to call Parasolid functions using Common LISP syntax. This manual is a guide to using KID.

Reporting Faults to Parasolid: A guide to generating and submitting the data required to report a fault to Parasolid Support for further investigation or make an enquiry.

XT Format Manual: A guide to the standard Parasolid XT file format.

Foreign Geometry User Guide: If you already have your own geometry model for use within your application, you can still use it with Parasolid. This manual explains how you can do this.

KI Programming Reference: For long-standing customers, this is the complete reference to the old interface to Parasolid, the Kernel Interface (KI). Although no longer actively developed, support is still provided for legacy code, though you should use the PK interface for any new application code you write. (Only available in HTML)
