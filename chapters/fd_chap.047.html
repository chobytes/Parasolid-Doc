46.1 Introduction

Parasolid provides specific functionality for some sheet modeling operations, and techniques that allow solid modeling functions to be used for sheet modeling. This chapter covers:

  • Creating sheet bodies - see Section 46.2.
  • Trimming sheet bodies - see Section 46.3.
  • Blending sheet bodies - see Section 46.4.
  • Thickening a sheet body to create a solid body - see Section 46.5.
  • Replacing the surface of a sheet body - see Section 46.6.1.

The boolean operation of sewing sheet bodies together is explained in Chapter 47, “Sewing and Knitting”.

Trimming operations are often performed using booleans (see Chapter 53, “Manifold Booleans”). However, there are circumstances where it can be more efficient to trim a multi-faced sheet body by first imprinting an entity (curves, faces or a plane) on to the body, and then trimming to the edges on the sheet body that were the result of the imprinting process. This is explained in Section 46.3 and Section 46.3.

Information on modeling operations which can be applied to wire bodies is contained in Chapter 45, “Wire Modeling”.

46.2 Creating sheet bodies

You can use PK_FACE_make_sheet_bodies to create one or more sheet bodies from a collection of faces. This function contains the following options:

 

Option

Description

 
allow_disjoint

Return disjoint components as either a single disjoint body or several bodies.

Default: PK_LOGICAL_false

 
make_from

Whether to make a sheet body (or bodies) from copies of the input faces, as illustrated in Figure 46-1, or from the original input faces themselves,as illustrated in Figure 46-2.

Default: PK_BODY_make_from_copy_c

 
transfer_attribs

Whether the sheet bodies inherit attributes from the topology used to create the sheet bodies.

Default: PK_LOGICAL_true

 
transfer_groups

Whether to transfer or copy specified groups to the new sheet body (or bodies).

Default: PK_GROUP_transfer_owning_c

 
track_edges

Whether to return edges from the resulting sheet bodies in the tracking data.

Default: PK_track_edges_no_c

 
track_vertices

Whether to return vertices from the resulting sheet bodies in the tracking data. Default: PK_track_vertices_no_c

 

Note: If you supply faces from different bodies to this function, you should ensure that they do not intersect. If the supplied faces intersect, PK_FACE_make_sheet_bodies may create an invalid body.

46.3 Trimming

Trimming is performed using PK_BODY_trim. To be successful, this must divide the body into at least two sets of faces. PK_BODY_trim takes the following arguments:

 

Argument

Description

body

The sheet body to be trimmed.

n_edges

edges

A list of edges defining the regions (face sets) on the sheet body.

n_faces

faces

A list of faces indicating the regions to be kept or discarded by the trimming operation.

keep

A boolean indicating which part of the body to keep and which part to discard.

  • PK_LOGICAL_true: keep any face sets which have a face in the faces list, and delete all other face sets.
  • PK_LOGICAL_false: delete any face sets which have a face in the faces list, and keep all other face sets.

The supplied edges and faces must all belong to the supplied body .

Should the trim operation fail, PK_BODY_trim_gap_analysis can be used to determine where the gaps of size less than the supplied tolerance exist, and the vertices bordering on these gaps are returned to enable corrective actions to be taken.

46.4 Blending sheet bodies

By using the solid modeling blend functions for applying, fixing, enquiring and checking blends it is possible to blend edges of sheet bodies which meet at the boundary of the sheet body. Parasolid constructs appropriate edge geometry at the ends of the blend. The blend type can be rolling-ball, variable-radius or chamfer.

Parasolid’s face-face blending functionality can be used to blend between disjoint and overlapping faces. In the following example face-face blends are applied sequentially to a set of sheet bodies, that have faces with taper angles, to form a single sheet body as follows:

46.4.1 Blending vertices on sheet bodies

To blend vertices on sheet bodies use PK_VERTEX_make_blend.

  • The vertices to be blended must have two edges meeting at them:
    • these two edges must share a common plane
    • if the face between the two edges has a surface then that surface must be planar

46.4.1.1 Limitations and restrictions

  • The positions calculated for each vertex are checked to make sure that none of the blends overlap with each other.
  • The validity of the resulting body is checked before the topology is changed or the new geometry is attached, to ensure that none of the edges intersect illegally.

 

Figure 46-8 The proposed blend would create an invalid body, so the blend is not created

 

Figure 46-9 By selecting both vertices the final body is valid and the blends can be created

  • If the radius is too large for any one of the vertices then none of the blending is implemented. Likewise if any one of the checks fail, then none of the blending is implemented.

It is permissible for two blends to be sufficiently large that they meet and exclude one of the original edges providing that the blends do not overlap. For example, it is possible to make a circle by blending the four vertices of a square.

 

Figure 46-10 Blending four vertices of a square to create a circle

  • It is also possible to blend a vertex with a radius which is equal to the distance to the next vertex. For example, if a vertex of a square sheet is blended with a radius equal to the length of the edges of the square, then two edges of the square are replaced by the blend.

 

Figure 46-11 Blending a vertex with a radius that is equal to the distance to the next vertex

If the local checking option is ON the body is checked after the blending has been carried out. In the case of a planar body the altered body is valid, but for a non-planar body local checking may return that the altered body is invalid.

46.6 Other modeling operations on sheet bodies

Providing that the relevant face of a sheet body has a surface attached to it, then the following operations can be carried out on it:

46.6.1 Replacing the surface of a sheet body

The surface of a sheet can be tweaked to a new surface, which must be of the same type as the original surface, using PK_BODY_embed_in_surf. During this operation the parameter space representation of the edges is maintained.

 

Note: This functionality does not offer support for facet geometry.

If the edge curve is of type PK_CLASS_spcurve or a constant parameter curve of the surface of the face then it is transferred to the new surface. All other types of edge curves are converted to SP-curves before being transferred to the new surface. During this conversion, edges may need to be split at surface discontinuities.
