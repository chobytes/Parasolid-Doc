Holds optional controls for removing a lattice from a region.

Used in:

PK_REGION_remove_lattice


The options provied are:

'attr_transfer'    Determines whether selected attributes are to be
                   moved to the region. It can take the following
                   values:

                   PK_ATTRIB_transfer_no_c
                       Move none of the attributes from the
                       frame to the region upon removal.
                       This is the default.

                   PK_ATTRIB_transfer_move_c
                       Move the selected attributes from the
                       frame to the region upon removal.

                   A move operation means that the attribute is first copied
                   to the region after which the attribute remaining on the
                   frame is deleted along with the frame.

                   Transfer of selected attributes is subject to the following
                   eligibility criteria:
                     o the attributes must be ownable by both regions and
                       frames
                     o The attributes must be of class 1, 4, 6 or 7. These are
                       the attribute classes which are not deleted upon change
                       events.
                   If either of these criteria is not met then
                   PK_ERROR_wrong_entity will be returned.

                   If 'attr_transfer' is set to PK_ATTRIB_transfer_move_c
                   and 'n_attdefs' = 0 then all attributes on the frame
                   which are eligible for transfer to the region will be
                   transferred.

                   This control does not affect the density system attributes.

'n_attdefs'        Length of 'attdefs'. This value is only meaningful when
                   'attr_transfer' is set to PK_ATTRIB_transfer_move_c.

'attdefs'          List of attribute definitions specifying the set of
                   attributes to copy or move to the frame.  This value is only
                   meaningful when 'attr_transfer' is set to
                   PK_ATTRIB_transfer_move_c.

'group_transfer'   Determines whether the region to which the frame is attached
                   is added to any eligible groups to which the frame belongs.
                   A group is eligible if it may contain both frames and
                   regions.

                   PK_remove_frame_group_remove_c
                        No additional processing is performed for groups.
                        The frame will be deleted and cease to be a member
                        of any of its respective groups.  This is the default
                        behaviour.

                   PK_remove_frame_group_replace_c
                        If the frame is a member of one or more groups but the
                        region to which it is attached is not a member of these
                        groups then the region will replace the frames in each
                        of those groups, subject to eligibilty (see above).
