This data structure contains operation status, the patterning
results and tracking information.

Used in:

PK_FACE_pattern_2
PK_FACE_pattern_2_r_f


Description of fields:



'result'           This status field indicates the overall state of
                   the patterning operations. Permitted values are :

                     PK_pattern_result_ok_c
                     PK_pattern_result_partial_c
                     PK_pattern_result_fail_c
                     PK_pattern_result_mixed_faces_c
                     PK_pattern_result_instanced_c
                     PK_pattern_result_invalid_set_c
                     PK_pattern_result_miss_geom_c



'n_patterns'       The length of the 'patterns' array.


'patterns'         The array of pattern instance results. If the 'result'
                   field indicates an error or a partial error, the individual
                   pattern result records (if returned) must be inspected.
                   If the 'result' field is PK_pattern_result_ok_c or
                   PK_pattern_result_partial_c, the number of elements in this
                   array will be the same as the number of pattern instances
                   requested, the value of 'n_transforms'.


'n_error_topols'   The number of topological entities in the 'error_topols'
                   array.


'error_topols'     If the 'result' field of this return structure is not
                   PK_pattern_result_ok_c, this array may contain topological
                   entities which are associated with the failure.


'tracking'         If the 'tracking_type' option is set to
                   PK_pattern_track_type_base_c, then this contains tracking
                   records created for merge, delete, split and derive events
                   involving base faces.

                   If, as a result of the patterning operation, two or more
                   base faces are merged, a tracking record with 'track' set
                   to PK_ENTITY_track_merge_c will be created. The
                   'original_entities' array will contain the base faces which
                   were merged and the 'transforms' responsible for the
                   merging. The 'product_entities' array will just contain the
                   new face. If just one base face merges with one or more
                   faces in a new pattern instance, a tracking record will not
                   be created.

                   If, as a result of the patterning operation, a base face is
                   split, a tracking record with 'track' set to
                   PK_ENTITY_track_split_c will be created. The
                   'original_entities' array will just contain the base face
                   which was split. The 'product_entities' will contain the
                   faces resulting from the splitting.

                   If, as a result of the patterning operation, a base face is
                   deleted, a tracking record with 'track' set to
                   PK_ENTITY_track_delete_c will be created. The
                   'original_entities' array will consist of the base face
                   which was deleted.

                   If, as a result of the patterning operation, a face is
                   produced by a combination of merging and splitting of two
                   or more base faces, a tracking record with 'track' set to
                   PK_ENTITY_track_derive_c will be created. The
                   'original_entities' array will contain the two or more base
                   faces involved and the 'transforms' responsible for the
                   merge. The 'product_entities' array will contain the faces
                   resulting from the event.


The function PK_FACE_pattern_2_r_f may be used to free associated memory,
including substructures.
