PK_TOPOL_imprint_frames_r_t    

struct PK_TOPOL_imprint_frames_r_s
    {
    int                 r_t_version;    --- version number of returns structure
    PK_ENTITY_track_r_t tracking;       --- tracking information
    };

typedef struct PK_TOPOL_imprint_frames_r_s PK_TOPOL_imprint_frames_r_t;



This data structure contains tracking information.

Used in:

PK_TOPOL_imprint_frames
PK_TOPOL_imprint_frames_r_f


Description of field:

'tracking'  Face tracking
            -------------

            For each frame that produces new facet faces, a record of type
            PK_ENTITY_track_create_c is created.  'original_entities' contains
            the frame and 'product_entities' contains the new faces.

            For each original face belonging to an imprinted frame's region,
            one or more records are created.  The number and type of records
            depends on the interaction of the original face with its adjacent
            frames, and also depends on region merging controlled by the
            'grow_void_regions' and 'grow_solid_regions' options, as follows:
                o If region merging causes there to be no surviving portions of
                  the original face, a record of type PK_ENTITY_track_delete_c
                  is created.  'original_entities' contains the original face.
                o If any surviving portions of the original face lie outside
                  all of its adjacent frames, a record of type
                  PK_ENTITY_track_derive_c is created.  'original_entities'
                  contains the original face and its original regions on both
                  sides, and 'product_entities' contains the face(s) lying
                  outside the adjacent frames.
                o If any surviving portions of the original face lie inside at
                  least one of its adjacent frames, one or more records of type
                  PK_ENTITY_track_derive_c are created.  In each record,
                  'original_entities' contains the original face and a set of
                  one or more overlapping frames adjacent to this face, and
                  'product_entities' contains the face(s) lying inside this set
                  of frames.  If 'original_entities' contains frames from only
                  one side of the face, the original region on the opposite
                  side is included in 'original_entities'.


            Region tracking
            ---------------

            For each set of overlapping frames (which may be a single frame if
            it is disjoint from all the others) that produces at least one new
            region, a record of type PK_ENTITY_track_create_c is created.
            'original_entities' contains the overlapping frames and
            'product_entities' contains one or more regions that were formed
            from this set of frames.

            For each original region that is split, a record of type
            PK_ENTITY_track_split_c is created.  'original_entities' contains
            the original region and 'product_entities' contains two or more
            split regions that lie outside all the original region's frames.

            For each original region that is deleted, a record of type
            PK_ENTITY_track_delete_c is created.  'original_entities' contains
            the deleted region.

            If the 'grow_void_regions' or 'grow_solid_regions' options cause
            regions to be merged, these merged regions are removed from the
            'product_entities' of any creation or splitting records, and the
            resultant region is added to the 'product_entities' of a new
            record.  'original_entities' in the new record contains all the
            'original_entities' from the old records, as well as any merged
            regions that were not created or split.  If any of the merged
            regions were previously split, the new record has type
            PK_ENTITY_track_derive_c, otherwise it has type
            PK_ENTITY_track_merge_c.  Any creation and splitting records that
            have lost all their 'product_entities' are deleted.  Any splitting
            records that have a single element remaining in 'product_entities'
            are deleted if 'product_entities' and 'original_entities' are the
            same, otherwise their type is changed to PK_ENTITY_track_derive_c.


The function PK_TOPOL_imprint_frames_r_f may be used to free associated memory.
