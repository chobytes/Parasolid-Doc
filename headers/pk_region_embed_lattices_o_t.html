typedef struct PK_REGION_embed_lattices_o_s PK_REGION_embed_lattices_o_t;



Holds optional controls for embedding lattices in the region.

Used in:

PK_REGION_embed_lattices


The options provided are:

'want_frames'      Controls whether an array of frames created by the
                   lattice embedding operation is returned.

'attr_transfer'    Determines whether selected attributes are to be moved
                   or copied to the frame. It can take the following values:

                   PK_ATTRIB_transfer_no_c
                       Move no attributes to the frame upon lattice embedding.
                       This is the default.

                   PK_ATTRIB_transfer_copy_c
                       Copy the selected attributes from the
                       region to the frame upon lattice embedding.

                   PK_ATTRIB_transfer_move_c
                       Move the selected attributes from the
                       region to the frame upon lattice embedding.

                   A copy operation means that independent instances of
                   the attribute will exist on both the region and the frame
                   after embedding the lattice.

                   A move operation means that the attribute is
                   first copied to the frame and then the instance remaining
                   on the region is deleted.

                   Transfer of selected attributes is subject to the following
                   eligibility criteria:
                     o The attributes must be ownable by both regions and
                       frames
                     o The attributes must be of class 1, 4, 6 or 7. These are
                       the attribute classes which are not deleted upon change
                       events.
                   If either of these criteria is not met then
                   PK_ERROR_wrong_entity will be returned.

                   If 'attr_transfer' is set to PK_ATTRIB_transfer_copy_c
                   or PK_ATTRIB_transfer_move_c and 'n_attdefs' = 0 then
                   all attributes on the region which are eligible (see above)
                   for transfer to the frame will be transferred.

                   This control does not affect the density system attributes.

'n_attdefs'        Length of 'attdefs'. This value is only meaningful when
                   'attr_transfer' is set to either PK_ATTRIB_transfer_copy_c
                   or PK_ATTRIB_transfer_move_c.

'attdefs'          List of attribute definitions specifying the set of
                   attributes to copy or move to the frame. This value is only
                   meaningful when 'attr_transfer' is set to either
                   PK_ATTRIB_transfer_copy_c or PK_ATTRIB_transfer_move_c.

'group_transfer'   Determines whether the frame created as part of lattice
                   embedding is added to all eligible groups where the region
                   is a member. A group is eligible if it may contain both
                   frames and regions. It may take the following values:

                   PK_embed_frame_group_no_c
                       No additional processing is performed for groups.
                       This is the default behaviour.

                   PK_embed_frame_group_add_c
                       Add the frame to all eligible groups (see above) where
                       the region is a member.

                   PK_embed_frame_group_replace_c
                       Replace the region with the frame in all
                       eligible groups (see above) where the region is a
                       member.

                   PK_embed_frame_group_remove_c
                       Remove the region from all groups where it is a
                       member.
