PK_PARTITION_set_guard    

PK_ERROR_code_t                   PK_PARTITION_set_guard
(
--- received arguments ---
PK_PARTITION_t                    partition,  --- partition
const PK_PARTITION_set_guard_o_t *options,    --- options structure

--- returned arguments ---
PK_PARTITION_set_guard_r_t *const returns     --- returns structure
)


This function sets the guard for the given 'partition' to its current state.


Specific Errors:
    PK_ERROR_rollback_not_started  Partitioned rollback is not active (MILD)
    PK_ERROR_bad_mark              Partition is at its initial pmark (MILD)


This function guards the current state of the 'partition', allowing a later
call to PK_PARTITION_goto_guard to restore it to that state.

Only one guard can be set for each partition and setting another guard will
replace the current guard. A guard cannot be set if the 'partition' is at its
initial pmark.

Making a pmark, rolling to a pmark or advancing a partition will unset any
guard in that partition. Making a mark or rolling to a mark will unset guards
in all partitions in the session.

This function should only be called when partitioned rollback has been started
with PK_DELTA_register_callbacks.
