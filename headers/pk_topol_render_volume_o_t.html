This structure contains optional controls for PK_TOPOL_render_volume.


Specific Errors:
    PK_ERROR_bad_tolerance          A tolerance is specified that is less than
                                    the relevant session precision. (MILD)
    PK_ERROR_bad_combination        An inconsistent combination of
                                    'min_facet_width' and 'max_facet_width'
                                    has been supplied. (MILD)

Used in:

PK_TOPOL_render_volume


Description of fields:

Scale factors
-------------

It is possible to associate a scale factor with a body such that all facets and
display primitives produced for a body or a region of that body are enlarged by
the supplied scale factor. The centre of scaling is the origin.

In order to supply scale factors for bodies it is necessary to fill in the
following fields:

'bodies_with_scales'
This field, of length 'n_bodies_with_scales', is an array of unique bodies
which are to have a scale factor applied. If this is empty no scaling will
occur. If this is not empty the parallel array 'scale_factors' must be
filled in.

'scale_factors'
This field must be of length 'n_bodies_with_scales'. Each PK_scale_factor_t
in this array is the scale associated with the corresponding body in
'bodies_with_scales'.

Note: If 'topol_transfs' are supplied any translation component of the
      transform will be applied after the scale factor is applied.

It is possible to shrink all the facets and display primitives returned by
providing an 'output_scale_factor' which will be used to scale all the returned
output with the scaling centre at the origin. No scaling is applied when the
value is PK_scale_factor_1x_c.


Faceting options
----------------

The tolerance values are each defined by a pair of PK_LOGICAL_t and double
values, the former indicating whether the double value has been set explicitly.
If no explicit tolerance value is provided, Parasolid is free to calculate its
own internal tolerance values. These are generally case specific, rather than
using specific fixed values.

The following tolerances can be set:

'min_facet_width'
In cases where refinement results in a facet being created that has a width
smaller than 'min_facet_width' Parasolid will disregard the other tolerances
so that the new facet will not be refined any further. Therefore facets may
exist with dimensions smaller than 'min_facet_width'.

Note: The 'min_facet_width' is ignored on topologies with facet geometry.

'max_facet_width'
This tolerance is the maximum allowed chord length of the side of a facet.

If both a 'min_facet_width' and 'max_facet_width' are specified, the
'min_facet_width' must be less than the 'max_facet_width' or
PK_ERROR_bad_combination will be returned.

'distance_tol'
This tolerance is the maximum allowed distance from the model geometry to its
associated facet, measured in model units.

'angular_tol'
This tolerance is the angular tolerance between the model geometry and its
associated facet, measured in radians. At edges this is an upper bound of the
sum of the angles between the chord and the curve tangents at either end, and
on a surface this is the maximum allowed deviation of the surface normal within
the facet.
