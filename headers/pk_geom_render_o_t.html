Controls for the PK_GEOM_render options structure.


Specific Errors:

    PK_ERROR_bad_parameter          'param' is PK_render_param_spaced_c or
                                    PK_render_param_spaced_free_c but 'param_u'
                                    and 'param_v' have not been specified
                                    OR
                                    'param' is PK_render_param_number_free_c
                                    but 'param_u_number' and 'param_v_number'
                                    have not been specified
                                    (MILD)
    PK_ERROR_bad_tolerance          'have_curve_chord_tol' is PK_LOGICAL_true
                                    and 'curve_chord_tol' is less than the
                                    value returned by PK_SESSION_ask_precision
                                    OR
                                    'have_curve_chord_max' is PK_LOGICAL_true
                                    and 'curve_chord_max' is less than the
                                    value returned by PK_SESSION_ask_precision
                                    OR
                                    'have_curve_chord_ang' is PK_LOGICAL_true
                                    and 'curve_chord_ang' is less than the
                                    value returned by
                                    PK_SESSION_ask_angle_precision
                                    (MILD)
    PK_ERROR_has_parent             'entities_with_scales' contains an entity
                                    of type PK_GEOM_t that is not an orphan
                                    geometry (MILD)
    PK_ERROR_wrong_entity           'entities_with_scales' contains an entity
                                    that is not of type PK_GEOM_t or PK_PART_t
                                    (MILD)

Used in:

PK_GEOM_render


This structure provides control over:

    - whether surface boundary and hatch lines are required
    - the rendering style to be used for B-curves
    - the rendering style to be used for lattices
    - the tolerances to be used when generating geometric information for GO
    - the scale by which to shrink the resulting representation

Each control tolerance is passed as a pair of values, consisting of a
PK_LOGICAL_t flag and a double precision value. If a specific tolerance
is required, the variable is set to a strictly positive value and the
associated flag set to PK_LOGICAL_true.

If Parasolid is to use its internal tolerance defaults, the associated flag is
set to PK_LOGICAL_false (and no use made of the associated tolerance argument).
This is the default state set by the PK_GEOM_render_o_m macro.

It is possible to associate a scale factor with an entity so that any geometry
of that entity is enlarged by the supplied scale factor in the rendering.
- If the geometry is construction (i.e. it would return an owning part if
  passed to PK_*_ask_part), then its owning part should be specified in
  'entities_with_scales'.
- If the geometry is orphan, the geometry should be specified in
  'entities_with_scales'

If a scale factor is specified for an assembly, it will only apply to
construction geometry directly attached to that assembly.

In order to supply scale factors for entities it is necessary to fill in the
following fields:

'n_entities_with_scales'
This is the number of entities with which a scale factor will be associated. If
this is 0, no scaling will occur.

'entities_with_scales'
This field must be of length 'n_entities_with_scales', it takes an array of
unique entities which are to have a scale factor applied. If this is empty no
scaling will occur. If this is not empty the parallel array 'scale_factors'
must be filled in. All elements of this array must be either a part or an
orphan geometry.

'scale_factors'
This field must be of length 'n_entities_with_scales', each PK_scale_factor_t
in this array is the scale associated with the corresponding entity in
'entities_with_scales'.

Note: If 'transfs' are supplied any translation component of the transform
will be applied to the geometries after they have been scaled.

The 'output_scale_factor' is the scale factor used to shrink the resulting
representation, with the origin used as the scaling centre. No scaling is
applied when the value is PK_scale_factor_1x_c.
