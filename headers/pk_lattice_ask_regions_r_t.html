struct PK_LATTICE_ask_regions_r_s
    {
    int             r_t_version;   --- version number of return structure
    int             n_regions;     --- number of regions
    PK_REGION_t    *regions;       --- the regions
    PK_LOGICAL_t   *senses;        --- the senses
    PK_FRAME_t     *frames;        --- the frames
    };
typedef struct PK_LATTICE_ask_regions_r_s PK_LATTICE_ask_regions_r_t;


A structure packaging the results of PK_LATTICE_ask_region.

Used in:

PK_LATTICE_ask_regions
PK_LATTICE_ask_regions_r_f


The structure has the following fields:

'n_regions'     The number of regions in which 'lattice' is embedded.

'regions'       The regions in which 'lattice' is embedded.

'senses'        The sense of the 'lattice' in each of the 'regions'.
                `senses[i]' is the sense of the lattice in `regions[i]'.
                PK_LOGICAL_true indicates that the interior of the 'lattice' is
                considered solid in the corresponding region. PK_LOGICAL_false
                indicates that the interior of the 'lattice' is considered void
                in the corresponding region. This will be set to NULL if
                'want_senses' was set to PK_LOGICAL_false, otherwise the array
                will be parallel to 'regions'. The sense can be
                reversed by calling the function PK_FRAME_reverse.

'frames'        The frames embedding the 'lattice' in each of the 'regions'.
                `frames[i]' embeds the 'lattice' in `regions[i]'. This will be
                set to NULL if 'want_frames' was set to PK_LOGICAL_false,
                otherwise the array will be parallel to 'regions'.

The function PK_LATTICE_ask_regions_r_f may be used to free associated memory.
