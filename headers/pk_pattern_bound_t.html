struct PK_pattern_bound_s
    {
    PK_LOGICAL_t           have_repetitions_i;
                            --- whether a bound has been provided in the ith
                            --- direction (PK_LOGICAL_true)
    int                    repetitions_i[2];
                            --- repetition bounds in the ith direction (0, 0)
    PK_LOGICAL_t           have_repetitions_j;
                            --- whether a bound has been provided in the jth
                            --- direction (PK_LOGICAL_true)
    int                    repetitions_j[2];
                            --- repetition bounds in the jth direction (0, 0)
    PK_LOGICAL_t           have_repetitions_k;
                            --- whether a bound has been provided in the kth
                            --- direction (PK_LOGICAL_true)
    int                    repetitions_k[2];
                            --- repetition bounds in the kth direction (0, 0)
    PK_pattern_trim_data_t trim_curves;
                            --- outline bounds for the ith and jth direction
                            --- (0, NULL, NULL, NULL, NULL, 0, NULL)
    PK_pattern_trim_type_t trim_type;
                            --- how to interpret the bounds of 'trim_curves'
                            --- (PK_pattern_trim_type_none_c)
    };
typedef struct PK_pattern_bound_s PK_pattern_bound_t;




This structure contains data on the bounds of a given pattern


Specific Errors:
    PK_ERROR_failed_to_bound        The combination of provided arguments
                                    would result in at least one invalid
                                    unbounded direction. (MILD)
    PK_ERROR_inconsistent_data      Inconsistent options specified as
                                    follows:
                                    - 'trim_type' has been set to
                                      PK_pattern_trim_type_none_c and
                                      'trim_curves' have been provided.
                                      (MILD)
                                    - 'trim_type' has been set to a value
                                      other than PK_pattern_trim_type_none_c
                                      and no 'trim_curves' have been provided.
                                      (MILD)

Used in:

PK_BODY_make_patterned_o_t
PK_LATTICE_make_patterned_o_t


Description of fields:



'have_repetitions_i'   Whether 'repetitions_i' is used. If this is set to
                       PK_LOGICAL_true then the pattern is bound by the
                       provided number of repetitions in the direction
                       corresponding to the 'i_axis' within the 'skewbox'.
                       If this is set to PK_LOGICAL_false, then
                       'repetitions_i' is ignored.


'repetitions_i'        Repetition bounds in the ith direction.


'have_repetitions_j'   Whether 'repetitions_j' is used. If this is set to
                       PK_LOGICAL_true then the pattern is bound by the
                       provided number of repetitions in the direction
                       corresponding to the 'j_axis' within the 'skewbox'.
                       If this is set to PK_LOGICAL_false, then
                       'repetitions_j' is ignored.


'repetitions_j'        Repetition bounds in the jth direction.


'have_repetitions_k'   Whether 'repetitions_k' is used. If this is set to
                       PK_LOGICAL_true then the pattern is bound by the
                       provided number of repetitions in the direction
                       corresponding to the 'k_axis' within the 'skewbox'.
                       If this is set to PK_LOGICAL_false, then
                       'repetitions_k' is ignored.


'repetitions_k'        Repetition bounds in the kth direction.


'trim_curves'          Loops of curves that bound the pattern in the ith and
                       jth pattern directions.


'trim_type'            Controls how 'trim_curves' data is interpreted to trim
                       the bounds.

Considering that the 'trim_curves' provides the bounding in the ith and jth
direction, the combination of the values provided for the fields must result
in at least one bound per direction, unless the pattern is periodic in that
direction.

If 'have_repetitions_i' and/or 'have_repetitions_j' is true and 'trim_type' is
set to a value other than PK_pattern_trim_type_none_c then both bounds will
apply, the pattern will have no more than the provided number of repetitions
and will be bounded by the 'trim_curves'.

For instance, if repetitions in all three directions are set to { -1, 1 }, and
their corresponding have_repetitions are set to PK_LOGICAL_true, then the
result has 27 duplicates of the original (i.e. 3x3x3). Alternatively, if
'trim_type' is not set to PK_pattern_trim_type_none_c, and
'have_repetitions_i' and 'have_repetitions_j' are set to PK_LOGICAL_false, then
the pattern will be bound in i and j by the 'trim_curves' and have a depth of 3
repetitions in k.
